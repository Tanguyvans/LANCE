## Tool catalog

### Graph tools (network model queries)
- **get_network_topology()** — Full topology (nodes + edges) as JSON
- **get_device_info(device_id)** — Device details + its neighbors
- **get_attack_surface()** — Devices with open ports/services
- **get_attack_paths()** — Multi-hop attack paths + pivot nodes
- **get_risk_scores()** — Risk scores per device (CVSS + exposure + centrality)

### Reconnaissance tools (real network commands)
- **arp_scan()** — Layer 2 ARP table: lists ALL devices on the local network with IP + MAC address. Fastest and most complete discovery method. No parameters needed.
- **nmap_discovery(target)** — Ping scan (-sn) to discover live hosts. Backup for arp_scan.
- **nmap_scan(target, ports?, skip_discovery?)** — nmap -sV scan (service/version detection). Scan ONE IP at a time. ALWAYS specify ports (e.g. ports="22,80,443,1883") — omitting ports triggers a slow 1000-port scan. Use skip_discovery=true if the host is up by ARP but nmap says "Host seems down".
- **ssh_audit(host, port?)** — SSH audit: ciphers, Terrapin, vulnerabilities. Default port 22. Only call if port 22 is confirmed open.
- **curl_headers(url)** — HTTP headers + body. Detects: missing security headers, directory listings ("Index of" in body), exposed sensitive files. Use on /backup/, /config/, /admin/ paths.
- **mqtt_listen(broker, topic?, count?, timeout?)** — Passive MQTT listener. topic="#" by default, count=10, timeout=5s. return_code=0 or 27 → anonymous access CONFIRMED. return_code=5 → auth required.
- **modbus_scan(target, skip_discovery?)** — nmap -sV -p 502. Port 502 open = Modbus TCP (no auth, HIGH severity). Use on PLC/SCADA/HMI devices.
- **nvd_lookup(query)** — Direct NIST NVD query (no cache). Prefer **cve_search** instead — it is faster and caches results. Use nvd_lookup only if cve_search is unavailable.

### Exploitation tools (controlled lab — real attacks allowed)
- **ssh_login(command_string)** — SSH login with credentials. Use sshpass: `sshpass -p admin ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null admin@IP 'id && cat /etc/iot/config.json'`. Returns command output as proof.
- **mysql_query(host, user, query)** — MySQL/MariaDB query. Try root with no password: mysql_query(host="IP", user="root", query="SHOW DATABASES; SELECT * FROM smartcity.users LIMIT 5;"). Returns query results.
- **telnet_connect(command_string)** — TCP connection via netcat: `echo quit | timeout 3 nc IP 23`. Returns banner/response.
- **ftp_list(url, user?)** — FTP listing: ftp_list("ftp://IP/"). Returns directory contents.
- **http_get(url)** — HTTP GET with full body. Use to retrieve actual file contents: http_get("http://IP/backup/db_backup.sql"), http_get("http://IP/.env").

### Physical attack tools (hardware — require physical lab access)
- **hackrf_capture(frequency, output_file, sample_rate?, num_samples?)** — HackRF One SDR for RF sniffing and replay. Covers Zigbee 802.15.4 (2.4 GHz), LoRaWAN EU868 (868 MHz), sub-GHz (433 MHz). Returns protocol-specific commands.
- **flipper_zero(protocol, action, frequency?)** — Flipper Zero multi-tool for sub-GHz replay/bruteforce, RFID 125 kHz cloning, NFC 13.56 MHz emulation, IR replay, GPIO/UART bridging.
- **proxmark3(protocol, action)** — Proxmark3 Easy for RFID/NFC badge cracking and cloning. Mifare Classic (darkside, hardnested), NTAG, DESFire, EM4100, HID Prox.
- **exploit_iot_kit(interface, device?, baudrate?)** — Hardware exploit kit for debug interface access (UART, JTAG, SWD), flash memory dumping (SPI, I2C), firmware extraction, and voltage glitching.

### Skill tools (IoT security knowledge)
- **list_skills()** — List available IoT security skills with descriptions and tags
- **load_skill(skill_name)** — Load a full skill document for detailed methodology. Use list_skills() first to discover available skills
- **search_knowledge(query, collection?, top_k?)** — Semantic search across CVE knowledge and skills
- **cve_search(query, top_k?)** — **Preferred CVE lookup.** Searches local cache first (fast), falls back to live NVD on miss and caches the result. Use this instead of nvd_lookup.
- **search_history(query, device_id?, top_k?)** — Search previous run findings for a device or vulnerability type
