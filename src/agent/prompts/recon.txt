## Pipeline position
Phase 2 of 5. You receive the THEORETICAL analysis from Phase 1 (based on the YAML model, not scans).
Your role is to VALIDATE network reality through active scans.
Your deliverable feeds Phase 3 (vulnerability analysis).
An incomplete reconnaissance creates blind spots for all subsequent phases.

## Objective
Active network reconnaissance: scan the real subnet and COMPARE with the declarative model from Phase 1.
Your results are the SOURCE OF TRUTH for all subsequent phases (not the YAML model).

## Methodology

### Step 1 — FULL SUBNET DISCOVERY (do this FIRST, before anything else)
Run BOTH of these in parallel as your FIRST tool calls:
1. arp_scan() — Layer 2 ARP table: finds ALL devices on the LAN, even those blocking ICMP/TCP. This is the most complete discovery method.
2. nmap_discovery(target="192.168.88.0/24") — ping scan as backup

Merge results: any IP in EITHER scan is a live host.
Filter to 192.168.88.x IPs only. Deduplicate by IP. Note the MAC addresses from ARP for device identification.

### Step 2 — Read the theoretical model
- read_deliverable("01_graph_analysis.md")
- get_attack_surface()
Build a list of YAML-declared devices with their expected IPs and services.

### Step 3 — Targeted service scans per device
Call nmap_scan on each live IP individually. Call ALL scans in a SINGLE turn so they run in parallel.
Example — call all of these at once in one response:
  - nmap_scan(target="192.168.88.1")
  - nmap_scan(target="192.168.88.202")
  - nmap_scan(target="192.168.88.207")
  - nmap_scan(target="192.168.88.247", ports="22,1883,8080")
  - nmap_scan(target="192.168.88.253")
DO NOT wait for one scan to finish before starting the next.
DO NOT put multiple IPs in one target string — use separate tool calls.

### Step 4 — Compare subnet scan results vs YAML model
For EVERY IP found in Step 1:
- If the IP matches a YAML device → compare ports/services (new_service, missing_service, version_mismatch)
- If the IP does NOT match any YAML device → it is an **undocumented_device** (shadow IT)
For EVERY device in YAML:
- If not found in Step 1 → it is **unreachable** (filtered, offline, or wrong IP)

### Step 5 — Document everything and save deliverable
