## Pipeline position
Phase 2 of 5. You receive the THEORETICAL analysis from Phase 1 (based on the YAML model, not scans).
Your role is to VALIDATE network reality through active scans.
Your deliverable feeds Phase 3 (vulnerability analysis).
An incomplete reconnaissance creates blind spots for all subsequent phases.

## Objective
Active network reconnaissance: scan the real subnet and COMPARE with the declarative model from Phase 1.
Your results are the SOURCE OF TRUTH for all subsequent phases (not the YAML model).

## Methodology
1. Read the Phase 1 deliverable: read_deliverable("01_graph_analysis.md") — this is the THEORETICAL model to validate
2. Check the DECLARED attack surface from the graph: get_attack_surface()
3. Run an nmap scan on subnet 192.168.88.0/24 to discover ACTUALLY open services
4. For each device in the attack surface, run a targeted nmap scan with expected ports
5. SYSTEMATICALLY compare nmap results with the graph model:
   - Service present in nmap but absent from model → **new_service** (discovery)
   - Service absent from nmap but present in model → **missing_service** (filtered or closed port)
   - Different version between nmap and model → **version_mismatch** (update or YAML error)
6. Document unreachable devices (timeout, filtering)
7. CLEARLY flag discrepancies: this is the key value of this phase
