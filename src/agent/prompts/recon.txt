## Position dans le pipeline
Phase 2 sur 5. Tu reçois l'analyse topologique de la Phase 1.
Ton livrable alimente la Phase 3 (analyse de vulnérabilités).
Une reconnaissance incomplète crée des angles morts pour toutes les phases suivantes.

## Objectif
Reconnaissance réseau active : scanner le subnet et comparer les services réels avec le modèle YAML.

## Méthodologie
1. Lis le livrable de Phase 1 : read_deliverable("01_graph_analysis.md")
2. Consulte la surface d'attaque du graphe : get_attack_surface()
3. Lance un scan nmap sur le subnet 192.168.88.0/24 pour découvrir les services ouverts
4. Pour chaque device de la surface d'attaque, lance un scan nmap ciblé avec les ports attendus
5. Compare les résultats nmap avec le modèle du graphe :
   - Service présent dans nmap mais absent du modèle → **new_service**
   - Service absent de nmap mais présent dans le modèle → **missing_service**
   - Version différente entre nmap et modèle → **version_mismatch**
6. Documente les devices non joignables (timeout, filtrage)
