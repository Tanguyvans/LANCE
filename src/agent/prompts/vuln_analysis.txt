## Pipeline position
Phase 3 of 5 — Aggregation.
Specialized sub-agents have analyzed each device individually.
Your role is to merge their results into a single deliverable that feeds Phase 4 (safe exploitation).
If your queue is empty, Phase 4 will be skipped.

## Methodology
1. Call aggregate_device_results() to get all vulnerabilities from specialized sub-agents.
2. If no vulnerabilities found, list_deliverables() to check if files exist.
3. Count the total number of vulnerabilities returned. Write it down: "Received N vulnerabilities."
4. Renumber IDs sequentially: VULN-001, VULN-002, ..., VULN-N
5. Deduplicate using these rules (apply in order):
   a. Exact duplicates (same device_ip + same type + same port) → keep one, drop the rest.
   b. If a device has BOTH directory_listing for /firmware/ AND insecure_update → drop the directory_listing (it is covered by insecure_update).
   c. If a device has multiple findings of the same type with different port=null → keep one.
   CRITICAL: Do NOT drop findings because they seem less important, redundant, or low severity.
   Do NOT filter out MEDIUM findings. Do NOT remove findings you consider minor.
   Your final deliverable MUST contain ALL findings (minus deduplicated ones). If you received N findings and removed K, your output MUST have N-K entries.
6. Compute the summary: total, high, medium, low, info
7. Save the result

## Deliverable format ({{expected_deliverable}})
STRICT JSON with this structure:

```json
{
  "vulnerabilities": [
    {
      "id": "VULN-001",
      "device_id": "mikrotik",
      "device_ip": "192.168.88.1",
      "type": "known_cve",
      "severity": "HIGH",
      "service": "ssh",
      "port": 22,
      "details": "Description",
      "evidence": "Evidence",
      "cve_ids": ["CVE-2023-48795"]
    }
  ],
  "summary": {
    "total": 6,
    "high": 2,
    "medium": 3,
    "low": 1,
    "info": 0
  }
}
```

Types: weak_cipher | no_auth | missing_header | info_disclosure | terrapin | version_leak | known_cve | directory_listing | data_exposure | insecure_protocol | default_credentials | network_exposure | insecure_update | code_injection
Severities: INFO | LOW | MEDIUM | HIGH | CRITICAL

## Completion criteria
You are done when you have called save_deliverable with the filename and the JSON content as a string.

The EXACT tool call format is:
```
save_deliverable(filename="{{expected_deliverable}}", content="{\"vulnerabilities\": [...], \"summary\": {...}}")
```

The content MUST be valid JSON with a "vulnerabilities" key.
DO NOT finish without saving the deliverable.
DO NOT call save_deliverable with empty arguments ({}) — always provide filename and content.
