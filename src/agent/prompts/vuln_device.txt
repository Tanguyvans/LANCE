<methodology>
## Méthodologie — OBLIGATOIRE pour chaque service du device

1. Lis le contexte des phases précédentes :
   - read_deliverable("01_graph_analysis.md") pour la topologie
   - read_deliverable("02_recon.md") pour les services découverts

2. Pour chaque service de {{device_id}}, applique les tests appropriés :

   **SSH** (si détecté) :
   - ssh_audit("{{device_ip}}", port)
   - Cherche : ciphers faibles, Terrapin (CVE-2023-48795), algos obsolètes

   **HTTP/HTTPS** (si détecté) :
   - curl_headers("http://{{device_ip}}:port")
   - Vérifie : X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security, Content-Security-Policy, Server version leak

   **MQTT** (si détecté) :
   - mqtt_listen("{{device_ip}}", timeout=5, count=3)
   - Teste l'accès anonyme

   **Ports custom** :
   - nmap_scan("{{device_ip}}", "port1,port2")

3. **NVD LOOKUP — OBLIGATOIRE** :
   Pour CHAQUE service avec une version connue, lance nvd_lookup(query) :
   - Construis une requête CPE ou keyword à partir du nom/version du service
   - Exemples : nvd_lookup("cpe:2.3:a:eclipse:mosquitto:2.0.11:*:*:*:*:*:*:*"), nvd_lookup("Dropbear SSH 2020.81")
   - **RÈGLE** : chaque CVE retournée avec CVSS >= 4.0 DOIT devenir une entrée "known_cve" dans ton livrable
   - Ne skip PAS cette étape, même si get_risk_scores() a déjà des CVEs

4. Corrèle avec les CVEs du graphe : get_risk_scores() et get_device_info("{{device_id}}")
</methodology>
