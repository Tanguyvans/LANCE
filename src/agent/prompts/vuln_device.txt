<methodology>
## Methodology — MANDATORY for each service on the device

1. Read context from previous phases:
   - read_deliverable("01_graph_analysis.md") for the topology
   - read_deliverable("02_recon.md") for discovered services

2. For each service on {{device_id}}, apply the appropriate tests:

   **SSH** (if detected):
   - FIRST check read_deliverable("02_recon.md") — if ssh_audit data for {{device_ip}} is already present, use it directly. Do NOT re-run ssh_audit in that case.
   - Only call ssh_audit("{{device_ip}}", port) if the data is missing from 02_recon.md.
   - Look for: weak ciphers, Terrapin (CVE-2023-48795), obsolete algorithms

   **HTTP/HTTPS** (if detected):
   - curl_headers("http://{{device_ip}}:port") — check security headers and server banner
   - MANDATORY: test for directory listing and exposed files:
     curl_headers("http://{{device_ip}}/backup/"), curl_headers("http://{{device_ip}}/config/"), curl_headers("http://{{device_ip}}/admin/")
     If response contains "Index of" or a file listing → type=directory_listing, severity=HIGH
     If a file exposes credentials or sensitive data → type=data_exposure, severity=HIGH

   **MQTT** (port 1883, or if device role is mqtt_broker):
   - ALWAYS call mqtt_listen("{{device_ip}}", topic="#", count=3, timeout=5) even if 1883 is not listed in services
   - CRITICAL exit code interpretation: return_code=27 (timeout) OR return_code=0 → anonymous access CONFIRMED → type=no_auth, severity=HIGH
   - Only return_code=5 means authentication is required (no finding)

   **Telnet** (for routers and gateways — device type=router or gateway):
   - ALWAYS run nmap_scan("{{device_ip}}", ports="23") to check for Telnet
   - If port 23 is open → type=insecure_protocol, severity=HIGH (Telnet exposes credentials in cleartext)

   **Custom ports**:
   - nmap_scan("{{device_ip}}", "port1,port2")

3. **NVD LOOKUP — MANDATORY**:
   For EACH service with a known version, run nvd_lookup(query):
   - Build a CPE or keyword query from the service name/version
   - Examples: nvd_lookup("cpe:2.3:a:eclipse:mosquitto:2.0.11:*:*:*:*:*:*:*"), nvd_lookup("OpenSSH 8.9"), nvd_lookup("nginx 1.18.0")
   - RULE: Use the software name and version as detected by nmap, NOT an assumed software.
   - **RULE**: every returned CVE with CVSS >= 4.0 MUST become a "known_cve" entry in your deliverable
   - Do NOT skip this step, even if get_risk_scores() already has CVEs
   - VALIDATION RULE: Before adding a CVE to your deliverable, verify:
     1. The affected software matches the software detected by nmap (not just the protocol)
     2. The affected version range includes the detected version
     If unsure, add the CVE with a note "version match unconfirmed" rather than fabricating.

4. Cross-reference with graph CVEs: get_risk_scores() and get_device_info("{{device_id}}")
</methodology>
